vite构建的vue3项目，svg文件的使用

2022-04-24
vite

  vite构建的项目中如何集中存放ui给的svg文件或者iconfont上的图标，然后优雅地使用呢？

1、最简单的方法

  使用vite构建的vue3项目,可以使用vite-vue3-svg-icon，不需要其他任何配置，也不需要额外的插件，利用vite的import(*?raw)实现资源的懒加载。

安装vite-vue3-svg-icon

1

yarn add vite-vue3-svg-icon@^1.0.1

全局引入组件

// main.ts
import svgIcon from 'vite-vue3-svg-icon'

createApp(App)
  .component('svg-icon', svgIcon)
  .mount('#app')

// svgIcon.vue

<template>
  <svg
    :viewBox="viewBox"
    aria-hidden="true"
    class="svgClass"
    v-html="svgContent"
    @click="clickHanle"
  ></svg>
</template>

<script setup lang="ts">
import { ref, watch, onMounted } from 'vue'

const props = defineProps({
  iconClass: {
    type: [String, Promise],
    default: ''
  },
  clickStop: {
    default: false
  }
})

const emit = defineEmits(['click'])
const clickHanle = e => {
  props.clickStop && e.stopPropagation()
  emit('click', e)
}

const viewBox = ref('0 0 1024 1024')
const svgContent = ref('')
let svgModuleDefault: any = ''
const reflashSvg = async() => {
  if(props.iconClass.then) {
    let svgModule: any =  await props.iconClass
    svgModuleDefault = svgModule.default
  } else {
    const svgFiles = import.meta.glob('SVG_PATH/*.svg', { as: 'raw' })
    const svgkey = Object.keys(svgFiles).find(key => new RegExp(`/${props.iconClass}.svg$`).test(key))
    svgModuleDefault = svgkey ? svgFiles[svgkey] : ''
  }
  if(!svgModuleDefault) {
    throw new Error('can not find svg file')
  } else {
    const svgEl:any = document.createElement('div')
    svgEl.innerHTML = svgModuleDefault
    viewBox.value = svgEl.children[0].getAttribute('viewBox') || '0 0 1024 1024'
    svgContent.value = svgEl.getElementsByTagName('svg')[0].innerHTML
  }
}

onMounted(reflashSvg)
watch(() => props.iconClass, reflashSvg)
</script>

<style>
.svgClass {
  height: 1em;
  overflow: hidden;
  vertical-align: -0.15em;
  width: 1em;
}
</style>

3、来自于雪碧图灵感的方法

  利用svg标签内可以使用use引用symbol定位svg内容的原理，在项目构建阶段把所有svg文件的内容，通过symbol拼在一起，挂在body下率先一次加载。使用的时候通过use引用，类似于古老的雪碧图的概念。   构建这样的雪碧图，使用webpack时，可以用svg-sprite-loader插件，在vite下，也有大神写出了实现类似功能的插件vite-plugin-svg-icons。

// vite.config.ts
import { createSvgIconsPlugin } from 'vite-plugin-svg-icons'
import path from 'path'

export default () => {
  return {
    plugins: [
      createSvgIconsPlugin({
        // 指定需要缓存的图标文件夹
        iconDirs: [path.resolve(process.cwd(), 'src/assets/svg')],
        // 指定symbolId格式
        symbolId: 'icon-[dir]-[name]',

        /**
         * 自定义插入位置
         * @default: body-last
         */
        inject?: 'body-last' | 'body-first'

        /**
         * custom dom id
         * @default: __svg__icons__dom__
         */
        customDomId: '__svg__icons__dom__',
      }),
    ],
  }
}

// svgIcon.vue
<template>
  <svg
    aria-hidden="true"
    class="svgClass"
    @click="clickHanle">
    <use :xlink:href="symbolId" />
  </svg>
</template>

<script setup lang="ts">
import { computed } from 'vue'
const props = defineProps({
  iconClass: {
    type: [String, Promise],
    default: ''
  },
  clickStop: {
    default: false
  }
})

const emit = defineEmits(['click'])

const clickHanle = e => {
  props.clickStop && e.stopPropagation()
  emit('click', e)
}

const symbolId = computed(() => `#icon-${props.iconClass}`)
</script>

<style>
.svgClass {
  height: 1em;
  overflow: hidden;
  vertical-align: -0.15em;
  width: 1em;
}
</style>
